# NGAPD

NGAPD 是一款面向独立游戏团队的 AI 原生敏捷项目管理系统。它将递归任务、任务级工作区、Agent 工具调用和过程知识沉淀组织为同一个领域模型。

项目的核心定义是：

> 任务 = 递归执行单元 + 单一责任边界 + AI 上下文边界 + 可追溯知识节点

## 已确认的产品边界

- 客户端支持 macOS、Windows；Web 服务端部署于 Linux。
- 项目规模主要面向少于 20 人的团队。
- 同一父任务下的直接子任务预计不超过 200 个，但该数值不是硬限制。
- 每个任务最多一个任务 Owner；任务工作区只有该 Owner 可以写入，其他项目成员只读。
- 任务依赖只能建立在同一父任务的直接子任务之间，并且不能成环。
- 管理员模式仅扩大 Web 端任务管理权限，不绕过工作区单写者权限。
- Agent 同时受 Web 任务权限、工作区权限和 Agent 确认策略约束。
- Agent 对当前任务内容和已连接工作区内的非破坏性修改可直接执行；其他写操作需要人工确认，破坏性操作始终需要确认。

## 设计原则

1. 能力角色与系统权限分离。
2. 每个任务只有一个责任人，不引入复杂的关注者、经办人和审批人模型。
3. 验收、复查和集成工作仍使用普通任务节点表达。
4. 任务图是权威业务结构；文件目录不承担任务层级关系。
5. Agent 与 Web UI 调用相同的领域服务，不维护第二套业务规则。
6. Wiki 是工作过程的可追溯投影，而不是独立维护的信息孤岛。
7. 优先构建模块化单体，避免过早拆分微服务。
