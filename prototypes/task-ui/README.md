# 平铺树状任务界面原型

## 要验证的假设

不依赖语义缩放或无限画布，也能通过“本级列表 + 当前详情 + 直接子任务 + 面包屑”稳定浏览深层任务树，并在同一层级展示 DAG 依赖。

## 范围

- 5 至 8 层任务深度，单父任务 200 个直接子任务。
- 虚拟项目根节点下的顶层依赖，以及任一父任务下的同级依赖。
- 普通、冲刺、里程碑三种只改变外观的展示类型。
- 搜索定位、筛选、面包屑、返回上一级和恢复原层级。
- 稀疏、中等和较密集的无环依赖。

## 不做

- 不做语义缩放、递归嵌套图、无限画布或自动任务排期。
- 不在原型中实现完整编辑、权限、评论与通知。
- 不为了动画效果牺牲键盘操作或 200 项浏览流畅性。

## 场景与数据

[dataset.json](fixtures/dataset.json) 是确定性数据生成规范。原型应生成稳定 Task Key，并保留父子关系与依赖的独立表达。

重点观察：

1. 从项目根定位到最深任务并逐级返回。
2. 在 200 个同级任务中搜索、筛选和滚动。
3. 切换任务后，本级列表、详情、子任务和面包屑保持一致。
4. DAG 方向明确，且不会把跨层任务误画成同级依赖。
5. 三种展示类型不改变可用操作。

## 退出标准

- 目标设备上可流畅浏览 200 个同级任务。
- 用户可从项目根定位到任意深层 Task Key，并稳定返回上一级。
- 层级、当前选择和依赖方向始终一致。
- 键盘可到达主要导航与列表操作，状态不只用颜色表达。
- 结果记录包含测量方式、设备信息和发现的交互问题。
