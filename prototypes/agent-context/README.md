# Agent 上下文原型

## 要验证的假设

系统可以先给 Agent 一份有来源、可分页、受权限约束的上下文清单，再由 Agent 渐进读取内容；工具授权不依赖模型是否正确理解工作区文件中的指令。

## 范围

- 当前用户的全部逻辑角色、自我介绍和用户级流程 Skill。
- 项目规则/Skill、当前任务统一正文、多层祖先和已完成 predecessor 摘要。
- 当前任务直接关注的一跳任务；关注链不递归展开。
- 工作区文本与图片元数据、上下文预算、优先级和截断原因。
- 恶意或冲突的工作区文本只能作为内容，不能扩大工具权限。
- Agent 提供摘要、用户提供摘要和系统基本字段兜底摘要三种来源。

## 不做

- NGAPD 服务端不调用外部 API、AI 或 LLM。
- 不评价具体模型的语言质量，不把 prompt 内容当授权来源。
- 不自动发现、搜索或读取其他用户的用户级工作区。

## 场景与数据

[context-sources.json](fixtures/context-sources.json) 定义候选来源、优先级、权限和预算，[workspace-injection.md](fixtures/workspace-injection.md) 用于验证提示注入不会改变工具授权。

重点检查：

1. 规则冲突时遵守固定优先级，并保留来源。
2. 超预算时优先保留安全规则、当前任务和必要祖先信息。
3. 不加载无关兄弟任务、其他用户 Skill 或递归关注链。
4. 图片只以元数据/明确引用进入清单，按需读取。
5. 摘要缺失时只生成确定性的基本字段一句话概括。

## 退出标准

- Agent 可通过清单渐进读取完成任务分析。
- 每条上下文均可解释来源、权限依据、优先级和是否被截断。
- 工作区内容不能授予工具能力、扩大读写范围或跳过人工确认。
- 同一输入和预算产生确定性的清单顺序与裁剪结果。
- 不发生外部模型或 API 调用。
