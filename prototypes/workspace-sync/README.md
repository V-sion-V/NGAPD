# 工作区租约与同步原型

## 要验证的假设

在用户级、项目级和任务级工作区上，同一时刻最多只有一个服务端认可的写入者；租约或基线版本失效后停止上传，并由仍有写资格的用户显式选择本地或服务端版本成为新的唯一事实。

## 范围

- 首次物化、文本修改、二进制图片、删除、重命名与空目录消失。
- 同一 Owner 的两台设备、两个 Project Admin、Owner 变化和客户端崩溃。
- 获取、续期、释放、到期和强制接管租约。
- `sync_version`、manifest 哈希、幂等提交和冲突选择。
- Windows/macOS 文件名、大小写、Unicode NFC、路径分隔符与保留名称。
- 排除 `.ngapd/`、`TASK.md` 和 `SUMMARY.md` 控制文件。

## 不做

- 不做增量块传输、实时协同编辑、后台静默合并或大文件优化。
- 不做最终安装器、自动更新、完整 UI 和生产级加密凭据存储。
- 不以文件监听事件作为正确性来源；提交前必须重新扫描并计算 SHA-256 manifest。

## 场景

以 [scenarios.json](fixtures/scenarios.json) 为执行清单，以 [base-manifest.json](fixtures/base-manifest.json) 为初始服务端事实。每个场景至少记录一次请求/响应、租约状态、基线版本、结果 manifest 与审计事件。

优先顺序：

1. 正常获取租约、修改、提交、续期和释放。
2. 第二设备争用、租约到期、崩溃恢复和旧持有者迟到提交。
3. 版本冲突后选择本地或服务端版本。
4. Owner/权限变化时存在未同步文件。
5. Windows/macOS 路径与名称差异。

## 退出标准

- 任一时刻每个工作区最多一个服务端认可的写入者。
- 冲突不会静默覆盖任一方，且选择前保留双方可识别版本。
- 合格写入者选择后只产生一个递增的服务端事实版本。
- 旧租约、旧基线或已失去写资格的客户端不能提交。
- 受保护路径不会进入用户内容 manifest。
- 相同夹具重复执行得到相同 manifest 哈希和状态转换。

## 输出

按 [结果模板](results/README.md) 记录结论。失败项必须区分协议问题、实现缺陷和环境限制，并指出是否需要调整 ADR-009 或正式同步契约。
